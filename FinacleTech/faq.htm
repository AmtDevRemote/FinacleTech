Frequently Asked Questions
Ques: Reports are not displayed for a date, although there is a message log for that date. What is the problem?
 
Ans: Check for any errors in the system log. If error says parsing exception, it means, message files are corrupt for any of the dates in between. All messages are stored in the Unicode format and rolled over date wise to generate message file for each day. If these message files are edited manually and saved, they may get corrupt if they are saved in a different format. Try accessing report for some other date range. If the reports are run properly, the problem lies only with that file. The reports data can’t be displayed from this corrupt file.
Ques: Even though I change the values through configuration panel, settings are not getting reflected?
 
Ans: After any change in the configuration setting, related to SMS service, changes will reflect only when the service is restarted. Use Start/Stop option of the SMS Service to restart the service.
